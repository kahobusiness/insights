# 产品七问

《产品七问》是一篇总纲。是我们接下来要谈论的许多问题的源头。作为产品设计师，以下七问，每条都可谓是灵魂拷问。要做出好的产品，我们需要给出深刻回答。

- 业务层面
    1. 是不是用户需要的？→ 用户是谁，解决什么问题
    2. 技术上能不能实现？→ 我们是谁，能否做好，成本如何
    3. 产品上应不应该做？→ 是否契合产品定位，该「取」还是「舍」
    4. 环境上允不允许做？→ 政策是否允许，市场趋势、公司战略如何
- 产品层面
    1. 是什么
- 运营层面
    1. 怎么做
- 团队层面
    1. 我们是谁


## 业务层

在业务层面，我们要对所做事情的 [用户价值]() ，有一个清晰的理解。

- 是不是用户需要的？ → 用户是谁，解决什么问题
- 技术上能不能实现？ → 我们是谁，能否做好，成本如何
- 产品上应不应该做？ → 是否契合产品定位，该「取」还是「舍」
- 环境上允不允许做？ → 政策是否允许，市场趋势、公司战略如何，是否应该当下就做

想清楚后，我们可以推理出一个[[业务目标]]，其描述是简单清晰的。同时，通过目标达成时的一些现象或场景的抽象和量化，我们可以设定出具体的[[业务指标]]。


## 产品层

在对业务层面有一个清晰的认知后，就可以开始在产品层面做定义——我们具体要做产品/工具/应用是什么。定义解决的问题、明确设计的理念、敲定功能的范围（[[归纳法与演绎法]]）、打磨使用体验，确保用户能通过工具解决问题。


## 运营层

运营层面包含品牌推广和服务设计两个方面。品牌推广谈论的如何找到潜在客户，找到合适的销售线索。服务设计谈论的是如何触达并将线索转化成用户，如何与用户保持沟通，如何通过反馈持续优化产品，如何通过活动等手段达成业务指标。

产品和运营工作是「做产品」的一体两面，必须两手抓，两手硬。它们共同营造出产品的「气质」与「调性」，好比 DNA 中的「双螺旋」，构成了一款产品的「基因」。
